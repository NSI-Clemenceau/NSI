Comment rendre une page web interactive ?

Remarque : n'hésitez pas à appliquer votre fichier CSS à cette page HTML pour la rendre plus attrayante.

1. Analyse du code source d'une page web interactive

On l'a vu...

  • Le langage HTML permet d'ajouter du contenu (texte, image,...) à une page web.
  • Le langage CSS permet de mettre en forme ce contenu (police, couleurs, mise en page,...).

Pour rendre interactive une page (clics, formulaires, contenu différencié,...), il faut introduire de nouveaux langages.

Les plus classiques sont...

  1. Javascript.
  2. PHP.

Toutefois, comme nous gardons l'objectif de finaliser le mini-projet du Choixpeau magique, en l'intégrant dans une page interactive, et que ce projet a été codé en Python, nous avons besoin d'une alternative moins connue, mais toutefois efficace : le langage Brython.

La première exigence du programme consiste à "examiner" le code source d'une page Web et d'y distinguer ce qui relève :

  • du contenu brut de la page (texte, images,...), géré par le HTML.
  • de la mise en forme de ce contenu par le CSS.
  • des événements associés aux différents composants graphiques (clic, survol de souris,...).

2. Ajouter des éléments dans la page à l'aide de Brython

  1. Etudier le début du tuto Brython afin de :
    • Repérer les lignes indispensables à ajouter à notre fichier html, afin de rendre Brython opérationnel.
    • Repérer la syntaxe permettant d'afficher un simple texte sur notre page.
  2. Mettre en oeuvre cette intégration de Brython et afficher un petit texte, comme "Hello World !" par exemple.
  3. Observer le résultat, puis noter vos remarques dans ce document de cours.

Remarques :

  • il n'est pas indispensable de lire le chapitre "Calculatrice"
  • comme on l'a vu, le script Brython s'exécute APRES l'affichage de la page HTML. Cela complique un peu l'organisation du contenu de la page. C'est pourquoi, je vous conseille de :
    • Soit créer / modifier l'ensemble de votre contenu par Brython.
    • Soit créer votre contenu en HTML, puis le modifier au besoin avec Brython. Il me semble que cette deuxième solution est plus simple.

L'objectif est de créer un paragraphe ci-dessous, en lui ajoutant un attribut id="texte_ephemere" pour l'identifier. Noter un texte minimaliste dans ce paragraphe puis faire en sorte que le clic sur le bouton1 modifie le contenu de ce paragraphe (changement de texte).

Maintenant, il faut utiliser la "valeur" du bouton2 pour afficher un texte lié à cette "valeur" : au clic sur le bouton2, le paragraphe précédent (texte_ephemere) doit afficher "La valeur du bouton 2 est 5", puisque 5 est la valeur récupérée dans l'attribut value du bouton2.

Remarque (aide) : l'attribut target, appliqué à l'évenement ev permet d'accéder au bouton cliqué. Pour atteindre la valeur de son attribut value, il suffit donc de saisir ev.target.value

Approfondissement : créer un compteur qui affiche et incrémente une valeur de 1 à chaque clic.

5. Que retenir ?

A minima...

  • Brython est un langage de programmation qui permet de rendre une page web interactive sans savoir programmer en Javascript. Il suffit de connaître Python.
  • Brython se charge de "traduire" notre code Python en un code Javascript, interprétable par un navigateur web. Il est donc naturellement moins efficace qu'une programmation directe en Javascript mais, pour la plupart des usages, cette différence est négligeable.
  • Pour utiliser Brython, il faut donc signaler dans la page HTML où trouver ce langage puis indiquer que l'on doit éxecuter ce code Brython à l'affichage de la page web (<body onload="brython()">).
  • Le code Brython est ensuite inclus entre deux balises <script></script>.
  • On peut ajouter des éléments HTML avec du code Brython.
  • On peut aussi interagir avec ces éléments HTML, en changeant leur contenu, leur style, leur affichage,...

Au mieux...

  • Savoir ajouter des éléments HTML avec du code Brython.
  • Savoir sélectionner des éléments HTML avec leur id ou leur class.
  • Savoir modifier un élément avec Brython
  • L'interactivité d'un bouton implique de créer une fonction qui s'éxecutera après un clic sur ce bouton.
  • Plus généralement, une fonction peut s'éxecuter après tout événement sur la page (affichage, clic, menu déroulant, saisie d'un champ de texte, survol d'un élément,...).
